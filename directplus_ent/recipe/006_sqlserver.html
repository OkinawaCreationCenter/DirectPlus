データベース性能向上に即効、ボトルネック解消レシピ

小森博之│2017.03.22(2019.03.04アップデート)│SQL Server

データベースのパフォーマンスをいかに向上させるか。システム管理者の皆さまを悩ませるこの課題に、今すぐ効くレシピをご紹介しましょう。パフォーマンス向上の決め手は、SSDの10～100倍ものスピードでストレージのボトルネックを解消する最新デバイス「NVDIMM-N」です。目的に合わせて使える構成モデルもご用意しています。

さあレシピをどうぞ！

メモリのスピードを備えたストレージ

「高性能サーバーにはSSD」―――こんな常識を覆すのが、メモリバスに直結する新しいタイプのストレージデバイス「NVDIMM-N」です。その特長は何と言ってもSSDの10倍～100倍という驚異的なI/O性能。NVDIMM-Nに対応しているHPE ProLiant Gen10サーバーなら、NVDIMM-Nをメモリスロットに挿すだけでそのパフォーマンスが今すぐ体感できます。Windows Server 2016からはNVDIMM-Nがネイティブサポートされ、さらに使いやすくなりました。

SQL Server 2017の性能を劇的にアップ

ファイルアクセスの高速化はすべてのアプリケーションで体験できますが、対応アプリケーションならもっと劇的な効果が得られます。その代表選手がMicrosoft SQL Server 2017。例えば新機能の「Directアクセスモード」を利用すれば、NVDIMM-NのIOPS性能はさらに10倍高速化できます。これはもう異次元の速さと言っていいでしょう。今最も強力で即効性のあるデータベースアクセラレーター、それがNVDIMM-Nです。

トランザクション処理も高速化

トランザクション処理では、その一貫性を保つためのログ処理がボトルネックになることがあります。SQL Server 2016からの新機能"Tail of Log"とNVDIMM-Nを使えばこの問題もあっという間に解決できます。NVDIMM-N上にログバッファーを生成しログ書き込みの回数を大幅に削減するとともに1回あたりの書込みも高速化。これによりトランザクション処理はNVMe SSDを使ったシステムの約2倍も高速化できます。

17万円で導入できる強力DBアクセラレーター

17万円ちょっとのデバイスをメモリスロットに挿すだけで、データベースのパフォーマンスを大幅に向上できるNVDIMM-N。高価なハードウェアや高度なSQLチューニングはもう不要です。今回はNVDIMM-Nを搭載した高性能データベースを手軽にご導入いただくために、SQL Server 2017の2つのエディションに対応したレシピをご用意しました。どちらも高性能で少コアのCPUを採用することでライセンス費用を抑えています。ぜひご検討ください。
